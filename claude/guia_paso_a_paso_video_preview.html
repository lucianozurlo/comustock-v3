Fase 1
Instalar Node.js 24 con nvm
1

Instalar nvm (Node Version Manager)

Terminal

Abrir una terminal y ejecutar:

# Descarga e instala nvm:
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.40.3/install.sh | bash

# En lugar de reiniciar la shell, cargar nvm manualmente:
\. "$HOME/.nvm/nvm.sh"
¿Ya tenés nvm? Saltá este paso. Verificá con nvm --version.
2

Instalar Node.js 24

Terminal

# Descarga e instala Node.js:
nvm install 24

# Verifica la versión de Node.js:
node -v   # Debería mostrar "v24.x.x"

# Verifica versión de npm:
npm -v    # Debería mostrar "11.x.x"
⚠️ Si ya tenés otra versión de Node. nvm permite tener múltiples versiones. Para que n8n siempre use la 24: nvm alias default 24
Fase 2
Instalar n8n
3

Instalar n8n globalmente con npm

Terminal

npm install -g n8n

Esperar a que termine (puede tardar 1-2 minutos). Verificar:

n8n --version   # Debería mostrar 1.x.x
Fase 3
Instalar FFmpeg
4

Instalar FFmpeg según tu sistema operativo

Terminal

Ubuntu / Debian:

sudo apt update
sudo apt install -y ffmpeg

macOS:

brew install ffmpeg
brew install coreutils   # necesario para el comando "timeout"

CentOS / Amazon Linux:

sudo dnf install -y epel-release
sudo dnf install -y https://download1.rpmfusion.org/free/el/rpmfusion-free-release-8.noarch.rpm
sudo dnf install -y ffmpeg

Cualquier Linux (binario estático, no necesita apt/dnf):

wget https://johnvansickle.com/ffmpeg/releases/ffmpeg-release-amd64-static.tar.xz
tar xf ffmpeg-release-amd64-static.tar.xz
sudo cp ffmpeg-*/ffmpeg ffmpeg-*/ffprobe /usr/local/bin/

Verificar que FFmpeg y ffprobe están disponibles

Terminal

ffmpeg -version   # Debe mostrar versión 4.4+
ffprobe -version  # Viene incluido con FFmpeg

Opción alternativa
Docker — n8n + FFmpeg en un contenedor
Usá esta opción en lugar de las Fases 1–4 si preferís Docker. Incluye n8n + FFmpeg en un solo contenedor. Node.js viene incluido dentro de la imagen de n8n.
A

Crear el Dockerfile

Terminal — en cualquier carpeta vacía

Crear una carpeta para el proyecto y el archivo Dockerfile:

# Crear carpeta del proyecto
mkdir ~/n8n-video-preview
cd ~/n8n-video-preview

# Crear el Dockerfile
cat > Dockerfile << 'EOF'
FROM n8nio/n8n:latest
USER root
RUN apk add --no-cache ffmpeg
USER node
EOF

Eso crea un archivo llamado Dockerfile (sin extensión) dentro de ~/n8n-video-preview/ con 4 líneas:

LíneaQué hace
FROM n8nio/n8n:latestUsa la imagen oficial de n8n (ya incluye Node.js)
USER rootCambia a root para poder instalar paquetes
RUN apk add --no-cache ffmpegInstala FFmpeg (la imagen base es Alpine Linux, usa apk)
USER nodeVuelve al usuario seguro "node"
B

Construir la imagen Docker

Terminal — dentro de ~/n8n-video-preview/

docker build -t n8n-ffmpeg .

El punto final (.) indica "usar el Dockerfile del directorio actual". Tarda 1-3 min la primera vez.

C

Levantar el contenedor

Terminal — dentro de ~/n8n-video-preview/

docker run -d \
  --name n8n-preview \
  -p 5678:5678 \
  -v n8n_data:/home/node/.n8n \
  -v /tmp:/tmp \
  -e N8N_ALLOW_EXEC=true \
  n8n-ffmpeg
FlagQué hace
-dCorre en background (detached)
--name n8n-previewNombre del contenedor (para referenciarlo después)
-p 5678:5678Expone el puerto 5678 del contenedor en localhost:5678
-v n8n_data:/home/node/.n8nVolumen persistente para datos de n8n (workflows, usuarios). Sobrevive a recrear el contenedor
-v /tmp:/tmpMonta /tmp del host para archivos temporales de FFmpeg
-e N8N_ALLOW_EXEC=trueHabilita los nodos Execute Command

Nodo 2: Guardar Video y Preparar Contexto

Canvas de n8n

Crear: Clic en el "+" que aparece a la derecha del nodo Webhook, escribir "Code", seleccionar "Code".

Renombrar: Doble-clic en el título del nodo (dice "Code") y escribir: Guardar Video y Preparar Contexto

Configurar error handling:

Icono de engranaje ⚙️ del nodo→Settings→On Error→Continue (using error output)

Pegar código: En el campo central del nodo (el editor de código JavaScript), borrar todo el contenido por defecto y pegar:

// ═══ NODO 2: Guardar Video y Preparar Contexto ═══

const fs = require('fs');
const item = items[0];
const binaryData = item.binary;

if (!binaryData || Object.keys(binaryData).length === 0) {
  throw new Error(
    'No binary data received. POST must be multipart/form-data with a video file.'
  );
}

const fileKey = Object.keys(binaryData)[0];
const file = binaryData[fileKey];

// Validar MIME type
const mime = (file.mimeType || '').toLowerCase();
const allowedMimes = [
  'video/mp4', 'video/quicktime', 'video/x-msvideo',
  'video/x-matroska', 'video/webm'
];
if (mime && !allowedMimes.some(m => mime.includes(m.split('/')[1]))) {
  throw new Error(
    'Tipo de archivo no soportado: ' + mime +
    '. Se esperaba un video (mp4, mov, avi, mkv, webm).'
  );
}

// Generar ID único para esta ejecución
const execId = 'vprev_' + Date.now() + '_'
  + Math.random().toString(36).substr(2, 6);
const basePath = '/tmp/' + execId;
const inputPath = basePath + '_input.mp4';

// Decodificar base64 y escribir a disco
const buffer = Buffer.from(file.data, 'base64');

// Validar tamaño máximo (default 500MB, ajustable)
const MAX_INPUT_MB = 500;
if (buffer.length > MAX_INPUT_MB * 1024 * 1024) {
  throw new Error(
    'Archivo demasiado grande: '
    + Math.round(buffer.length / 1024 / 1024)
    + 'MB. Máximo: ' + MAX_INPUT_MB + 'MB.'
  );
}

fs.writeFileSync(inputPath, buffer);

const stat = fs.statSync(inputPath);
if (stat.size === 0) {
  throw new Error(
    'El archivo escrito tiene 0 bytes. '
    + 'Verificar que el upload llegó correctamente.'
  );
}

return [{
  json: {
    execId,
    fileKey,
    inputPath,
    outputPath: basePath + '_output.mp4',
    concatListPath: basePath + '_concat.txt',
    basePath,
    mimeType: file.mimeType || 'video/mp4',
    originalName: file.fileName || 'input.mp4',
    inputSizeBytes: stat.size
  }
}];

Cerrar el nodo.

Nodo 3: Obtener Duracion

Canvas de n8n

Crear: Clic en el "+" a la derecha de "Guardar Video y Preparar Contexto", buscar "Execute Command", seleccionar.

Renombrar: Obtener Duracion

On Error: ⚙️ → Settings → On Error → Continue (using error output)

Comando: En el campo "Command", pegar exactamente esto (incluido el = al inicio):

=timeout 30 ffprobe -v quiet -show_entries format=duration -of csv=p=0 "{{ $json.inputPath }}"
⚠️ El = al inicio es obligatorio. Le indica a n8n que el campo es una expresión que debe evaluar (para reemplazar {{ $json.inputPath }} por la ruta real). Sin el =, ejecutaría el texto literal.

Nodo 4: Calcular Timestamps

Canvas de n8n

Crear: "+" → Code

Renombrar: Calcular Timestamps

On Error: ⚙️ → Settings → On Error → Continue (using error output)

Pegar código: Borrar el contenido default del editor y pegar:

// ═══ NODO 4: Calcular Timestamps ═══

const item = items[0];
const stdout = (item.json.stdout || '').trim();
const durationRaw = parseFloat(stdout);

if (isNaN(durationRaw) || durationRaw <= 0) {
  throw new Error(
    'ffprobe no pudo leer la duración. stdout: "' + stdout + '". '
    + 'Verificar: 1) FFmpeg instalado y en PATH, '
    + '2) archivo MP4 válido, 3) N8N_ALLOW_EXEC=true'
  );
}

// ─── CONFIG (modificar aquí para ajustar) ───────────
const TOTAL_PREVIEW  = 10;    // segundos totales del preview
const MAX_CLIPS      = 3;     // máximo de clips
const OUTPUT_WIDTH   = 350;   // px ancho de salida
const MIN_CLIP_DUR   = 0.5;   // mínimo clipDur
const FFMPEG_TIMEOUT = 120;   // timeout por comando FFmpeg (seg)
// ────────────────────────────────────────────────────

const D = durationRaw;
let N = MAX_CLIPS;
let clipDur = TOTAL_PREVIEW / N;

while (N > 1 && D < N * clipDur) {
  N--;
  clipDur = TOTAL_PREVIEW / N;
}

if (D <= clipDur) {
  N = 1;
  clipDur = D - 0.05;
  if (clipDur <= 0) clipDur = D;
}

if (clipDur < MIN_CLIP_DUR && D >= MIN_CLIP_DUR) clipDur = MIN_CLIP_DUR;
if (clipDur > D) clipDur = D;

const segment = D / N;
const timestamps = [];

for (let i = 0; i < N; i++) {
  let start = segment * i + (segment / 2) - (clipDur / 2);
  if (start < 0) start = 0;
  if (start + clipDur > D) {
    start = D - clipDur - 0.05;
  }
  if (start < 0) start = 0;

  timestamps.push({
    index: i,
    start:    parseFloat(start.toFixed(3)),
    duration: parseFloat(clipDur.toFixed(3)),
    clipPath: item.json.basePath + '_clip' + i + '.mp4'
  });
}

const ffmpegClipCmds = timestamps.map(t =>
  'timeout ' + FFMPEG_TIMEOUT
  + ' ffmpeg -y'
  + ' -ss ' + t.start
  + ' -i "' + item.json.inputPath + '"'
  + ' -t ' + t.duration
  + ' -vf "scale=' + OUTPUT_WIDTH + ':-2"'
  + ' -an'
  + ' -c:v libx264 -preset fast -crf 23'
  + ' -movflags +faststart'
  + ' "' + t.clipPath + '"'
  + ' 2>&1'
);

const concatLines = timestamps.map(
  t => "file '" + t.clipPath + "'"
);
const concatContent = concatLines.join('\n');

const concatCmd =
  'timeout ' + FFMPEG_TIMEOUT
  + ' ffmpeg -y'
  + ' -f concat -safe 0'
  + ' -i "' + item.json.concatListPath + '"'
  + ' -c copy -an'
  + ' -movflags +faststart'
  + ' "' + item.json.outputPath + '"'
  + ' 2>&1';

const clipPaths = timestamps.map(
  t => '"' + t.clipPath + '"'
).join(' ');
const cleanupCmd =
  'rm -f'
  + ' "' + item.json.inputPath + '"'
  + ' "' + item.json.outputPath + '"'
  + ' "' + item.json.concatListPath + '"'
  + ' ' + clipPaths
  + ' && echo CLEANUP_OK';

return [{
  json: {
    ...item.json,
    duration:       D,
    numClips:       N,
    clipDuration:   parseFloat(clipDur.toFixed(3)),
    outputWidth:    OUTPUT_WIDTH,
    timestamps,
    ffmpegClipCmds,
    concatContent,
    concatCmd,
    cleanupCmd
  }
}];

Nodo E1: Capturar Error

Canvas de n8n

Crear: Clic en una zona vacía del canvas (lejos de la línea principal), buscar "Code".

Renombrar: Capturar Error

NO configurar On Error (este nodo ES el handler, si falla, no tiene sentido redirigir).

Pegar código:

// ═══ Capturar Error ═══

const item = items[0];

let errorMsg = 'Error desconocido en el workflow';
if (item.json && item.json.error) {
  const e = item.json.error;
  if (typeof e === 'string') errorMsg = e;
  else if (e.message) errorMsg = e.message;
  else errorMsg = JSON.stringify(e);
}

console.error('[VideoPreview][ERROR]', errorMsg);

const cleanupCmd = item.json && item.json.cleanupCmd;
const inputPath  = item.json && item.json.inputPath;

let cmdToRun = null;
if (cleanupCmd) {
  cmdToRun = cleanupCmd;
} else if (inputPath) {
  cmdToRun = 'rm -f "' + inputPath
    + '" && echo PARTIAL_CLEANUP_OK';
}

if (cmdToRun) {
  try {
    const { execSync } = require('child_process');
    execSync(cmdToRun, { timeout: 5000 });
  } catch(e) {
    console.warn(
      '[VideoPreview] Cleanup en error handler falló:',
      e.message
    );
  }
}

return [{
  json: {
    success: false,
    error: errorMsg,
    timestamp: new Date().toISOString()
  }
}];

Conectar las ramas de error

Canvas de n8n

Cada nodo con "On Error: Continue (using error output)" tiene dos salidas: la salida de arriba es ✅ éxito (main[0]), la salida de abajo es ❌ error (main[1]).

Las conexiones de éxito (main[0]) ya se crearon automáticamente al usar el "+" de cada nodo. Ahora hay que conectar las ramas de error:

Cómo conectar: Arrastrá desde el punto de salida inferior (rojo/naranja) de cada nodo hacia la entrada del nodo "Capturar Error".

Conectar la salida de error (main[1]) de estos 10 nodos → Capturar Error:

#NodoSalida error → va a
2Guardar Video y Preparar Contexto→ Capturar Error
3Obtener Duracion→ Capturar Error
4Calcular Timestamps→ Capturar Error
5Extraer Clips FFmpeg→ Capturar Error
6Validar Clips→ Capturar Error
7Escribir Lista Concat→ Capturar Error
8Concatenar y Escalar→ Capturar Error
9Validar Concat→ Capturar Error
10Leer Preview MP4→ Capturar Error

Caso especial — Limpiar Temporales:

El nodo 11 (Limpiar Temporales) tiene ambas ramas (éxito y error) conectadas al mismo destino: Responder con Preview. Esto es porque si la limpieza falla, el preview ya fue generado y el cliente debe recibirlo igual.

Testear cada uno

Terminal

Script para testear todos de una vez:

#!/bin/bash
for f in /tmp/test_0.5s.mp4 /tmp/test_5s.mp4 /tmp/test_10s.mp4 /tmp/test_60s.mp4 /tmp/test_600s.mp4; do
  echo "=== Testing: $f ==="
  HTTP=$(curl -s -X POST \
    http://localhost:5678/webhook/video-preview \
    -F "file=@$f" \
    -o /tmp/prev_out.mp4 \
    -w "%{http_code}")

  if [ "$HTTP" = "200" ]; then
    W=$(ffprobe -v quiet -select_streams v:0 -show_entries stream=width -of csv=p=0 /tmp/prev_out.mp4)
    D=$(ffprobe -v quiet -show_entries format=duration -of csv=p=0 /tmp/prev_out.mp4)
    A=$(ffprobe -v quiet -select_streams a -show_entries stream=codec_type -of csv=p=0 /tmp/prev_out.mp4)
    echo "  HTTP: $HTTP | Width: ${W}px | Duration: ${D}s | Audio: ${A:-NONE}"
  else
    echo "  HTTP: $HTTP (error)"
  fi
  echo ""
done

Fase 9
Configurar para producción
1

Crear archivo .env para n8n

Terminal

Crear el archivo de configuración:

mkdir -p ~/.n8n
cat > ~/.n8n/.env << 'EOF'
N8N_ALLOW_EXEC=true
N8N_PORT=5678
N8N_PAYLOAD_SIZE_MAX=512
N8N_LOG_LEVEL=info
N8N_LOG_OUTPUT=console,file
N8N_LOG_FILE_LOCATION=/var/log/n8n/
EXECUTIONS_DATA_SAVE_ON_ERROR=all
EXECUTIONS_DATA_SAVE_ON_SUCCESS=none
EOF
⚠️ N8N_RUNNERS_ENABLED — Asegurate de que esta variable NO esté presente o esté en false. Si está en true, los Code nodes corren aislados y require('fs') falla.
2

Crear servicio systemd (Linux — para que n8n arranque automáticamente)

Terminal

sudo cat > /etc/systemd/system/n8n.service << 'EOF'
[Unit]
Description=n8n workflow automation
After=network.target

[Service]
Type=simple
User=$USER
EnvironmentFile=/home/$USER/.n8n/.env
ExecStart=$(which n8n) start
Restart=on-failure
RestartSec=5

[Install]
WantedBy=multi-user.target
EOF

sudo systemctl daemon-reload
sudo systemctl enable n8n
sudo systemctl start n8n

Verificar: sudo systemctl status n8n
